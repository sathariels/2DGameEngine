Design Decisions

üîÑ Fixed Timestep Game Loop

Decision: Implemented a fixed timestep loop with interpolation for rendering.

Why: Ensures deterministic physics and gameplay regardless of framerate, crucial for consistent game behavior across different hardware.

üñ•Ô∏è OpenGL Rendering

Decision: Used OpenGL with modern shader-based rendering pipeline.

Why: Provides hardware acceleration and flexibility for effects while maintaining cross-platform compatibility.

üíª C++ Core with Lua Scripting

Decision: Engine core in C++ with Lua scripting for game logic.

Why: Combines performance of C++ for engine systems with flexibility of Lua for rapid game development and modding.

üì¶ Component-Based Architecture

Decision: Entity-Component-System (ECS) pattern for game objects.

Why: Promotes code reusability, improves cache performance, and makes complex game logic more manageable.
